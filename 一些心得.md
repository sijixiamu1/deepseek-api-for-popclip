## 开发小插件

### 产生需求


我觉得可以平时多观察生活，想想有哪些可以自动化的操作，比如说我就是频繁浏览网页，发现很多英文不认识，
但是目前的翻译插件要么就是翻译全部网页，要么是机翻，要么太重，要么 api 太贵（gpt claude）
所以我觉得可以开发一个简单的插件

### 明确技术栈

这里可以问 ai，例如 gpt 这种，不要问豆包，很垃圾，让他给你生成需要的技术栈和教学

### 学习技术

这里就是把技术拆分成模块，**这里的学习可以和项目实践一起进行，先有一个初步的了解，之后再让 ai 给你生成项目代码，结合你的项目学习**
在 roadmap 网站上看一下对应的流程图，例如前端开发，就要学 html，css，和 js，一些库或者插件可以去**官网**查阅
然后找对应的教学，一定要边学边捣鼓一点代码

### 实践

claude写这些小众的库或者软件的时候可能会出错，**最好将官方的 api 喂给他，或者喂一个现成的项目**并且写一个 skill 文件

具体写项目的时候模块规划一定要清晰，然后再逐个击破，像我的小项目就包括了网页请求和 popclip 的显示

下一步就是调试，这里踩了不少坑，例如没有提前告诉 aiapi 的格式，他又有幻觉觉得自己是对的，没有提前对这个库有一点点了解，想着全靠 ai，还是不行的...

---
说了这么多，其实最重要的就是拆分问题，问 ai，查官网。。。
